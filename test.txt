








 reactDOM служит для связывания реакта с дом деревом
для порталов в реакте, создается отдельный компонент

const Portal = (props) => {
  const node = document.createElement('div');
  document.body.appendChild/Div(node);
  вшенаписаные констарции помоают разметстиь портал на странице
  return reactDOM.createPortal(props.children, node)
}
принимает два арумента, 1-й child - что хотим поместить (props.children), и 2-й - куда хотим поместить child (node)
следовательно, мы будем помещать props.children в элемент node
и уже в верстве, испольуем следкод:
<Portal><Msg/ - это элемент, который содержит аму верстку с необх эелментом></Portal>
такие порталы, располоаются в конце страницы в отдельном div-е
как же работают обработчики собыйти с такими порталами
если навесить обработчки события на форму (скрин из видоса) - то он будет работать даже на вынесенном элементе (скрин) - даже, пусть он рендерится не в самой форме судя по верстке (скрин), хотя он в реакте назодится в одном компоненте => события, сгенерированого извнутри портала будет распростояняться и на родителя (контекст реакта)
